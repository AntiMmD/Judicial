Search API

مستندات API جستجو و فیلتر در قوانین، مواد و تبصره‌ها

معرفی

این API برای جستجوی متنی در مدل Law استفاده می‌شود. جستجو روی قوانین (Law)، مواد (Article) و تبصره‌ها (Note) انجام می‌شود.

تمام endpointهای این اپ با پیشوند /search/ در دسترس هستند.

فلوی پیشنهادی فرانت:
  1. کاربر عبارت را جستجو می‌کند → بدون پارامتر type همه انواع نمایش داده می‌شوند.
  2. کاربر یک تب (مثلاً ماده) را انتخاب می‌کند → همان عبارت با type=Article ارسال می‌شود.
  3. فیلد facets همیشه تعداد نتایج هر نوع را نشان می‌دهد تا کاربر بین تب‌ها جابه‌جا شود.
لیست Endpointها
  • GET /search/ - جستجو در قوانین، مواد و تبصره‌ها
1. جستجو

جستجوی متنی در فیلدهای عنوان، خلاصه‌ها، متن اصلی، کد و مرجع تصویب. نتایج بر اساس relevance (مرتبط‌بودن)، اولویت و بازدید مرتب می‌شوند.

دسترسی: عمومی (بدون نیاز به احراز هویت)
Endpoint

GET /search/
Query Parameters
پارامتر نوع اجباری توضیح
q string خیر عبارت جستجو. در صورت خالی بودن، نتایج بر اساس اولویت مرتب می‌شوند.
type string خیر فیلتر نوع: Law، Article یا Note
category string خیر فیلتر دسته‌بندی قانون (مثلاً Regulation، Bylaw)
page integer خیر شماره صفحه (پیش‌فرض: 1)
page_size integer خیر تعداد نتایج در هر صفحه (پیش‌فرض: 20، حداکثر: 50)
مقادیر مجاز type
  • Law — قانون
  • Article — ماده
  • Note — تبصره
مقادیر مجاز category
  • Law
  • Agreement
  • Approval
  • Bylaw
  • Charter
  • Circular
  • Convention
  • Instruction
  • Procedure
  • Regulation
  • Statutes
نمونه درخواست — جستجوی کلی

GET /search/?q=قانون+مدنی
نمونه درخواست — فیلتر روی مواد

GET /search/?q=قانون+مدنی&type=Article
نمونه درخواست — فیلتر دسته‌بندی

GET /search/?q=قانون&type=Law&category=Regulation&page=1&page_size=20

توضیح فیلدهای پاسخ
فیلد توضیح
query عبارت جستجوی ارسال‌شده
type_filter نوع فیلترشده (یا null اگر همه انواع)
category_filter دسته‌بندی فیلترشده (یا null)
facets تعداد نتایج هر نوع برای همان q (بدون توجه به type)
relevance امتیاز مرتبط‌بودن — تطابق در عنوان بالاترین امتیاز را دارد
parent_title عنوان قانون والد (برای ماده و تبصره)
