Jobs API

مستندات API مربوط به مدیریت شغل‌ها (Job)

معرفی

این API برای مدیریت لیست شغل‌ها در سیستم استفاده می‌شود. هر کاربر می‌تواند یک شغل داشته باشد که از جدول Job انتخاب می‌شود.

تمام endpointهای این اپ با پیشوند /jobs/ در دسترس هستند.

دسترسی: تمام endpointهای این بخش فقط برای Admin در دسترس هستند.

لیست Endpointها
  • GET /jobs/ - دریافت لیست شغل‌ها
  • POST /jobs/ - ایجاد شغل جدید
  • GET /jobs/{id}/ - دریافت یک شغل
  • PUT /jobs/{id}/ - ویرایش کامل شغل
  • PATCH /jobs/{id}/ - ویرایش بخشی از شغل
1. دریافت لیست شغل‌ها

لیست تمام شغل‌های ثبت‌شده در سیستم.

Endpoint


GET /jobs/
نمونه پاسخ


[

{

“id”: 1,

“occupation”: “Engineer”,

“description”: “Software engineer”,

“is_active”: true

},

{

“id”: 2,

“occupation”: “Teacher”,

“description”: “School teacher”,

“is_active”: true

}

]
2. ایجاد شغل

ایجاد یک شغل جدید در سیستم.

دسترسی: Admin
Endpoint


POST /jobs/
نمونه درخواست


{

“occupation”: “Engineer”,

“description”: “Software engineer”,

“is_active”: true

}
نمونه پاسخ


{

“id”: 3,

“occupation”: “Engineer”,

“description”: “Software engineer”,

“is_active”: true

}
3. دریافت یک شغل

دریافت اطلاعات یک شغل بر اساس شناسه.

Endpoint


GET /jobs/{id}/
نمونه پاسخ


{

“id”: 1,

“occupation”: “Engineer”,

“description”: “Software engineer”,

“is_active”: true

}
4. ویرایش کامل شغل
Endpoint


PUT /jobs/{id}/
نمونه درخواست


{

“occupation”: “Senior Engineer”,

“description”: “Senior software engineer”,

“is_active”: true

}
5. ویرایش جزئی شغل
Endpoint


PATCH /jobs/{id}/
نمونه درخواست


{

“is_active”: false

}
6. دسترسی‌ها
Endpoint Permission
GET /jobs/ IsAdminUser
POST /jobs/ IsAdminUser
GET /jobs/{id} IsAdminUser
PUT /jobs/{id} IsAdminUser
PATCH /jobs/{id} IsAdminUser
7. ساختار آدرس‌ها

در فایل اصلی پروژه:



path(‘jobs/’, include(‘job.urls’))

بنابراین تمام endpointها با پیشوند /jobs/ در دسترس هستند.
