Accounts API

مستندات کامل API احراز هویت و مدیریت کاربران

معرفی

این API برای ثبت‌نام کاربران، احراز هویت مبتنی بر OTP و دریافت JWT Token استفاده می‌شود.

تمام endpointهای این اپ با پیشوند /auth/ در دسترس هستند.

نکته: کاربران می‌توانند با OTP یا با شماره تلفن و رمز عبور وارد سیستم شوند.
لیست Endpointها
  • POST /auth/sign-up - ثبت‌نام کاربر
  • GET /auth/me/ - دریافت اطلاعات کاربر جاری
  • PATCH /auth/me/ - ویرایش اطلاعات کاربر
  • POST /auth/request-otp/ - درخواست کد OTP
  • POST /auth/verify-otp/ - تایید OTP
  • POST /auth/login/ - ورود با رمز عبور
  • POST /auth/token/refresh/ - تمدید توکن
1. ثبت‌نام کاربر

ایجاد کاربر جدید در سیستم. بعد از ثبت‌نام، توکن‌های JWT نیز برگردانده می‌شوند.

دسترسی: عمومی (AllowAny)
Endpoint

POST /auth/sign-up
نمونه درخواست

{
  "phonenumber": "09123456789",
  "password": "strongpassword",
  "email": "user@example.com",
  "first_name": "Ali",
  "last_name": "Ahmadi",
  "job": 1
}
نمونه پاسخ

{
  "refresh": "jwt_refresh_token",
  "access": "jwt_access_token"
}
2. دریافت اطلاعات کاربر

دریافت اطلاعات کاربر لاگین‌شده.

دسترسی: نیازمند احراز هویت
Endpoint

GET /auth/me/
نمونه پاسخ

{
  "phonenumber": "09123456789",
  "email": "user@example.com",
  "first_name": "Ali",
  "last_name": "Ahmadi",
  "job": 1,
  "job_detail": {
    "occupation": "Engineer",
    "description": "Software Engineer"
  }
}
3. ویرایش اطلاعات کاربر

کاربر می‌تواند اطلاعات پروفایل خود را ویرایش کند.

Endpoint

PATCH /auth/me/
نمونه درخواست

{
  "first_name": "Ali",
  "last_name": "Karimi"
}
4. درخواست کد OTP

ارسال پیامک حاوی کد تایید برای شماره تلفن.

دسترسی: عمومی (AllowAny)
Endpoint

POST /auth/request-otp/
نمونه درخواست

{
  "phonenumber": "09123456789"
}
نمونه پاسخ

{
  "message": "OTP sent"
}
5. تایید کد OTP

بررسی OTP ارسال شده و در صورت صحیح بودن، توکن‌های JWT صادر می‌شود. اگر کاربر وجود نداشته باشد، به صورت خودکار ساخته می‌شود.

دسترسی: عمومی (AllowAny)
Endpoint

POST /auth/verify-otp/
نمونه درخواست

{
  "phonenumber": "09123456789",
  "otp": "123456"
}
نمونه پاسخ

{
  "refresh": "jwt_refresh_token",
  "access": "jwt_access_token"
}
6. ورود با رمز عبور

ورود کاربر با استفاده از شماره تلفن و رمز عبور.

Endpoint

POST /auth/login/
نمونه درخواست

{
  "phonenumber": "09123456789",
  "password": "your_password"
}
نمونه پاسخ

{
  "refresh": "jwt_refresh_token",
  "access": "jwt_access_token"
}
7. تمدید توکن

با استفاده از refresh token یک access token جدید دریافت می‌شود.

Endpoint

POST /auth/token/refresh/
نمونه درخواست

{
  "refresh": "your_refresh_token"
}
8. دسترسی‌ها
Endpoint Permission
/auth/sign-up AllowAny
/auth/request-otp AllowAny
/auth/verify-otp AllowAny
/auth/login AllowAny
/auth/me Authenticated User
9. ساختار آدرس‌ها

در فایل اصلی پروژه: 


path('auth/', include('Accounts.urls'))

بنابراین تمام endpointها با پیشوند /auth/ در دسترس هستند.
